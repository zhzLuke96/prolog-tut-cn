# 风格指南

## 1. 统一风格框架

### 1.1 两条路线，明确分界

教程保留两套入门风格，但在 README 和 SUMMARY.md 中明确标注推荐路径：

- **「从零开始」系列** → 零基础路线。保留学术翻译风（严谨、系统、逐步讲解），适合无编程背景读者。精炼冗长段落，但保持系统性。
- **「Hello Prolog」系列** → 程序员快速入门路线。保留口语调侃风（幽默、工程导向、互动式），适合有编程经验读者。缩短与前者重复的基础概念，专注工程实践。

### 1.2 交叉引用规则

- 「从零开始」的章节作者不应提及「Hello Prolog」内容（保持封闭完整性）
- 「Hello Prolog」可在章节开头用一句话指引读者去「从零开始」查阅更详细的概念解释
- 示例：
  > 关于联合（unification）的详细逐歩解释，请参阅「从零开始」chapter11。
- SUMMARY.md 中两条路线的章节标题前加 emoji 或标签区分：
  - 📘 从零开始学 Prolog
  - 🛠️ Hello Prolog：程序员实战

### 1.3 拼接章节（如 c4 DCG）写作策略

当两个系列覆盖同一主题时：
- 「从零开始」负责概念建立（what + why）
- 「Hello Prolog」负责工程应用（how + when + tradeoffs）
- 不在两套教程中完整重复同一概念的全过程

## 2. 代码规范

### 2.1 Prolog 方言选择

- **主力方言**：SWI-Prolog 8.x+（当前最活跃的 Prolog 实现，生态最丰富）
- **ISO Prolog**：作为可移植子集使用（基础的 facts/rules/lists/arithmetic）
- **非 SWI 特性必须标注**：使用 SWI 特有谓词时，在代码行末注释 `% SWI-Prolog only`
- **不在正文中详细对比**方言差异（如不出现"Turbo Prolog 是这样，SWI 是那样"的大段对比）
- **特殊场景例外**：SWI 不支持某些第 3 方方言特性时，作为"扩展阅读"附注，不干扰主线

### 2.2 代码声明头

每个 `.pl` 文件：
- 首行声明编码 `:- encoding(utf8).`（涉及中文内容时必须）
- 次行声明头：

```prolog
% SWI-Prolog: tested on version 8.x
```

或

```prolog
% ISO Prolog: portable across implementations
```

### 2.3 命名规范

| 场景 | 规范 | 示例 |
|------|------|------|
| 教学示例——人物 | 英文名 | `john`, `mary`, `tom` |
| 教学示例——中文场景 | 拼音（ASCII） | `zhangxueyou`, `wanfei` |
| 变量 | 有意义命名 | `Name`, `Age`, `SortedList` |
| 单字母变量仅限教程 | 逐步讲解中可使用 | `X`, `Y`, `Z` 标注"未知值" |
| 谓词命名 | snake_case | `quick_sort/2`, `find_all/3` |
| 匿名变量 | 使用 `_` 而非 `_VarName` | `member(X, L)` 而非 `member(X, _L)` |

### 2.4 注释规范

```prolog
% 单行注释描述意图，而非复述代码
% 句子以句号结尾

/*
 * 多行注释用于：
 * 1. 文件头部说明
 * 2. 复杂算法的概览
 * 3. 参考文献
 */

/** <examples>  % SWI-Prolog 文档注释，仅用于可执行示例
?- member(a, [a,b,c]).
?- member(X, [a,b,c]).
*/
```

### 2.5 禁止的代码写法

| 禁止 | 替代 |
|------|------|
| `not/1` | `\\+/1`（ISO 标准） |
| `get0/1` | `get_char/1` |
| 无类型检查的算术 | `number(X)` + `is/2` |
| `last(P,_)` 中 P 未绑定 | 先实例化再调用 |
| 覆盖标准库谓词 | 自定义谓词加前缀（如 `my_member/2`） |

## 3. 写作指导

### 3.1 术语统一表

| 使用术语 | 避免 | 说明 |
|----------|------|------|
| 事实 / 规则 / 查询 | 断言 / 子句 | 中文初学者更熟悉前者 |
| 联合 | 合一 / 匹配 | "联合"反映 unification 语义 |
| 回溯 | 倒退 / 回退 | 既定译法 |
| 截断 | 剪枝 / Cut | "截断"对应 `!` 操作符语义 |
| 谓词 | 过程 / 函数 | Prolog 术语 |
| 子句 | 语句 / 分句 | clause 的既定译法 |
| 自由变量 | 未绑定变量 | free variable 的统一译法 |
| 约束编程 | 限制编程 | CLP 的中文名称 |

### 3.2 示例案例统一

整个项目使用**一套贯通案例**，避免每章重新引入新案例：

**核心案例**（跨多个章节出现）：
- `friend/2`（人际关系）— 事实/规则/查询入门
- `find_nani/0` 游戏 — 递归/回溯/列表的驱动案例
- `mortal/1`（苏格拉底）— 逻辑推理入门
- 汉诺塔 / 八皇后 — 递归典型问题

**引入新案例规则**：
- 新案例必须在前 3 句内说明"这个案例的目的是教什么"
- 案例不超过 1 个章节范围，跨章节复用时必须添加引用说明
- Hello Prolog 可以引入工程案例（如 HTTP server、CLP 调度），但不应重复从零开始的经典例子

### 3.3 代码块标注规范

```markdown
```prolog
% 所有 Prolog 代码块标注为 prolog
% 不使用 js、swipl 或其他
```

针对多文件的展示：

```markdown
\`\`\`prolog
% file: hello.pl
:- initialization(main).
main :- write("Hello").
\`\`\`
```

### 3.4 对话语气控制

- 「从零开始」：使用"我们"/"读者"。避免"你"直接指代。
  - 对：`现在我们来观察递归的终止条件。`
  - 错：`你可以看到递归的终止条件。`
- 「Hello Prolog」：使用"你"。允许口语化表达和有控制的幽默。
  - 对：`看到了吗？屁股后面还回了 true。`
  - 错：`我们可以看到查询结果返回了 true。`
  - 限制：幽默不应精确性，不对新手造成理解歧义。

### 3.5 代码验证规则

所有 `.pl` 文件必须通过自动化验证：

```bash
swipl -g "load_files([filename]),halt"
```

在 CI 中添加：

```yaml
- name: Verify Prolog code
  run: for f in code/**/*.pl; do swipl -q -g "load_files(["$f"]),halt" || exit 1; done
```

### 3.6 可运行性要求

每段代码必须满足：
1. 可在 SWI-Prolog 8.x 直接加载 `[filename].` 无 error
2. 代码中给出的查询示例必须真实可运行并产生所述输出
3. 不完整的代码（仅展示概念片段）必须标注 `% 代码片段，不可直接运行`
4. 使用 `/** <examples>` 注释标注可运行的查询示例

## 4. Markdown 格式规范

### 4.1 代码中的特殊字符转义

**必须转义**：
| 字符 | 场景 | 正确写法 |
|------|------|----------|
| `*` | 代码中表示乘法 | `\*` 或在代码块中正常写 `*`（代码块内不渲染） |
| `_` | 谓词名中的下划线 | 代码块内正常（不会渲染为斜体） |

**规则**：
- 代码块 `\`\`\`prolog` 内的 `*` / `_` 不会被 Markdown 渲染，无需转义
- 代码块**外**的 `*` 用于强调时用 `\*` 转义
- 如果代码中 `*` 被渲染损坏（如 chapter13），检查是否被移出了代码块

### 4.2 章节头文件规范

```markdown
# 章节标题

> 一句话概括本章节内容。不超过 50 字。

## 子标题
```

### 4.3 图片管理

- 图片统一放在 `src/tut/img/` 或 `src/hello-pl/img/`
- 引用方式：`![](img/filename.png)`
- 禁止使用 `<img>` HTML tag（GitBook 可识别，HonKit 可能不兼容）
- 禁止引用外部 URL 图片（全部本地化）
- 文件名规则：`{章节编号}_{序号}.png`（如 `c3i1.png`）

### 4.4 代码和输出分离

```
```prolog
?- member(X, [a,b,c]).
X = a
```
```

所有查询和输出在同一个代码块内，以 `?-` 开头标记查询，输出直接跟在查询下方，无前缀。查询和输出之间不用分隔线。

### 4.5 链接规范

- 内部链接：相对路径 `[chapter1](chapter1.md)`
- 外部链接：必须注明是否需代理 `[SWISH](https://swish.swi-prolog.org/)（需科学上网）`
- 不添加 tracking parameters

## 5. 内容边界

### 5.1 该写的内容

- SWI-Prolog 标准库的使用和讲解
- 工程实践（编译、调试、测试、模块化）
- CLP(FD)、DCG、高阶谓词等现代特性
- 与 Python/Rust 等语言的交互（pyswip、SWI-Prolog FFI）
- 调试技巧（`guitracer`、`spy/1`、`trace/0`）
- 常见错误和 FAQ
- 性能注意事项（choice point、tail recursion、tabling）

### 5.2 不该写的内容

- 已停更 5 年以上 Prolog 实现（Turbo Prolog、PDC Prolog、Visual Prolog）的详细安装和使用说明（一句话提及即可）
- 与 Prolog 无关的哲学讨论（除非与教程主线直接相关）
- 第三方教程的大段引用（可标注参考链接）
- 未经验证 / 不可运行的代码示例
- 需要付费软件才能运行的内容

### 5.3 时效性规则

- 所有工具推荐（编辑器、插件、在线服务）必须标注最后验证日期
- 超过 2 年未更新的推荐需要添加警告标记
- 每年至少一次检查外部链接有效性

### 5.4 章节间衔接规则

- 同一系列的章末必须有"下一章预告"（1 句话）
- 同一系列的章首必须有"上章回顾"（2-3 句话）
- 不同系列之间在 SUMMARY.md 中有过渡说明，但正文中不串系列
