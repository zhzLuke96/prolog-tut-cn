# 模块化

> 一句话概括：用 `module/2` 和 `use_module/1` 把 Prolog 代码组织成可复用、命名隔离的单元。

> 前置：c1 | 难度：★★ | 后续：c8

## 模块基础

写大了就需要组织代码。SWI-Prolog 的模块系统提供命名空间隔离——一个模块内定义的谓词不会污染其他模块。

### 定义模块

文件开头声明：

```prolog
%% file: geometry.pl
:- module(geometry, [area/3, perimeter/3]).
```

`module/2` 第一个参数是模块名，第二个是导出列表。这个文件定义 `geometry` 模块，只导出 `area/3` 和 `perimeter/3`。

```prolog
%% geometry.pl 完整内容
:- module(geometry, [area/3, perimeter/3]).

area(circle, R, A) :- A is pi * R * R.
area(rect, (W, H), A) :- A is W * H.

perimeter(circle, R, P) :- P is 2 * pi * R.
perimeter(rect, (W, H), P) :- P is 2 * (W + H).
```

### 使用模块

```prolog
%% file: main.pl
:- use_module(geometry).

main :-
    area(circle, 5, A),
    format("Area: ~f~n", [A]).
```

`use_module/1` 导入默认加载 `geometry.pl`，在当前模块可见其导出的谓词。

### 模块查找路径

SWI-Prolog 默认搜索当前目录和 library 路径。可以用 `file_search_path/2` 注册：

```prolog
:- multifile user:file_search_path/2.
user:file_search_path(my_lib, '/path/to/my/libs').
:- use_module(my_lib(geometry)).
```

或者直接指定路径：

```prolog
:- use_module('/home/user/prolog/geometry.pl').
```

## 模块可见性控制

### export 声明

除了 `module/2` 的导出列表，模块内部可以用 `:- export` 追加：

```prolog
:- module(utils, []).

:- export([reverse/2, flatten/2]).
reverse([], []).
reverse([H|T], R) :- reverse(T, Rev), append(Rev, [H], R).
```

### import 控制

只导入特定谓词：

```prolog
:- use_module(geometry, [area/3]).
```

这样只导入 `area/3`，`perimeter/3` 不可见。

### 模块限定的调用

即使不 `use_module`，也可以通过模块前缀调用：

```prolog
?- geometry:area(circle, 3, A).
A = 28.274333882308138.
```

这在调试时很有用——临时访问未导入的谓词。

### 运算符导出

模块也可以导出运算符：

```prolog
:- module(arith, [op(500, yfx, @), (@)/2]).

:- op(500, yfx, @).

X @ Y :- X + Y.
```

使用者加载后就能用 `@` 运算符。

## 创建可发布的 Prolog Library

写一个完整 library 的步骤：

**1. 目录结构**

```
my_lib/
  pack.pl         % 包描述（可选）
  logic.pl        % 核心模块
  io.pl           % I/O 模块
```

**2. pack.pl（发布用）**

```prolog
name('my_lib').
version('1.0.0').
title('My Prolog Library').
author('Your Name', 'email@example.com').
home('https://github.com/you/my_lib').
```

**3. 模块内引用其他模块**

```prolog
%% logic.pl
:- module(logic, [solve/2]).
:- use_module(io).

solve(X, Y) :-
    read_input(X),
    compute(X, Y),
    output(Y).
```

**4. 安装为 pack**

```bash
$ swipl pack_install /path/to/my_lib
```

或上传到 GitHub，别人就能：

```bash
$ swipl pack_install yourname/my_lib
```

## 模块的注意事项

### 动态谓词

如果模块内用 `dynamic`，需要在模块内声明：

```prolog
:- module(db, [add/1, list/1]).

:- dynamic entry/1.

add(X) :- assert(entry(X)).
list(X) :- entry(X).
```

### 多文件模块

一个模块可以跨文件：

```prolog
%% tools.pl
:- module(tools, [foo/1]).
:- multifile tools:bar/1.

%% tools_extra.pl
:- module(tools_extra, []).
:- multifile tools:bar/1.
tools:bar(baz).
```

### 模块和 meta-call

Meta-call 需要模块上下文：

```prolog
call_in_module(Mod, Goal) :-
    Mod:Goal.
```

`maplist/2` 等高阶谓词自动处理模块上下文。

## 常见反模式

**反模式 1：一个巨型文件即所有**

把全部代码写成一个文件，靠空行分隔。导致命名冲突、难以测试。

**反模式 2：过度模块化**

每个谓词一个文件。引入大量 `use_module/1`，依赖图复杂。

**反模式 3：不导出、全黑盒**

什么都不导出，其他模块都无法使用。或者反过来——导出所有内部帮助谓词。

建议：按功能域分组，一个文件 50-200 行，导出接口谓词，内部帮助谓词藏起来。

## 总结

模块系统让 Prolog 项目从脚本集进化成真正的工程。关键点：

- `module/2` 定义模块 + 导出列表
- `use_module/1` 加载依赖
- 支持选择性导入、模块限定调用、运算符导出
- Pack 系统可发布可复用库
