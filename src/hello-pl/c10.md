# 单元测试

> 一句话概括：用 `plunit` 把 Prolog 谓词包在 `begin_tests/end_tests` 里，`run_tests` 一键验证。

> 前置：c3, c7 | 难度：★★ | 后续：—

## plunit 基础

SWI-Prolog 内置 `plunit` 测试框架，类似 xUnit。基本结构：

```prolog
:- begin_tests(math).
    test(add) :- assert(1 + 1 =:= 2).
    test(sub, [fail]) :- 1 - 1 =:= 3.
:- end_tests(math).
```

保存为 `test_math.pl`，加载后运行：

```prolog
?- run_tests.
% PL-Unit: math .. done
% 2 tests passed
true.
```

### test/1 和 test/3

`test(+Name)` — 基本测试，Body 必须成功。

`test(+Name, +Options)` — 带选项：

- `fail` — 测试预期失败
- `true` — 测试预期成功（默认）
- `nondet` — 允许多个解
- `forall(G)` — 为 G 的每个解运行测试
- `blocked` — 标记为阻塞（跳过）

```prolog
:- begin_tests(list).

    test(reverse) :-
        reverse([1,2,3], [3,2,1]).

    test(member_success) :-
        member(a, [a,b,c]).

    test(member_fail, [fail]) :-
        member(x, []).

    test(member_all, [nondet]) :-
        member(X, [1,2,3]).

:- end_tests(list).
```

## 编写测试谓词

### 结合之前章节

以 c6 的阶乘为例：

```prolog
:- begin_tests(factorial).

    test(base) :-
        factorial_tail(0, 1).

    test(small) :-
        factorial_tail(3, 6).

    test(large, [time(0.1)]) :-
        factorial_tail(1000, _).

    test(negative, [fail]) :-
        factorial_tail(-1, _).

    test(edge_zero) :-
        factorial_tail(0, 1).

:- end_tests(factorial).
```

### forall 批量测试

```prolog
:- begin_tests(double).

    test(double_all, [forall(between(1, 100, N))]) :-
        D is N * 2,
        D =:= N + N.

:- end_tests(double).
```

### setup / cleanup

```prolog
:- dynamic db/1.

:- begin_tests(db).

    setup :-
        assert(db(a)),
        assert(db(b)).

    cleanup :-
        retractall(db(_)).

    test(query_all, [set(Xs = [a,b])]) :-
        findall(X, db(X), Xs).

:- end_tests(db).
```

`setup` 和 `cleanup` 在每个 test 前后运行。

## run_tests 自动化

### 运行全部测试

```prolog
:- run_tests.

## 参考

- [The Power of Prolog — Testing](https://www.metalevel.at/prolog/testing)
- SWI-Prolog manual: plunit
```

放在文件末尾，加载时自动测试。

### 运行特定测试集

```prolog
?- run_tests(math).
% PL-Unit: math .. done
```

### 在 make 中集成

```prolog
test_all :-
    format("Running all tests...~n"),
    run_tests.
```

然后用 `swipl -q -t test_all -s test_suite.pl` 调用。

### 持续集成

配合 GitHub Actions：

```yaml
# .github/workflows/ci.yml
- name: Run Prolog tests
  run: swipl -q -t "run_tests" -s tests/suite.pl
```

## 实战：测试之前章节的谓词

写一个测试套件，统一验证各章节核心谓词：

```prolog
%% 引入各模块
:- use_module('../code/hello/fib').
:- use_module('../drafts/hello-pl/geometry').

%% 测试套件
:- begin_tests(hello_pl_suite).

clear_m :- retractall(m(_)).

    %% c1: Hello world
    test(hello_world) :-
        with_output_to(string(S), hello),
        S == "Hello World!\n".

    %% c6: Fibonacci with tabling
    test(fib_base) :-
        fib(0, 1), fib(1, 1).
    test(fib_small) :-
        fib(10, 89).
    test(fib_large, [time(1)]) :-
        fib(1000, _).

    %% c7: Module - geometry
    test(area_circle) :-
        geometry:area(circle, 3, A),
        abs(A - 28.274) < 0.001.
    test(area_rect) :-
        geometry:area(rect, 3, 4, 12).

    %% c8: Web server (仅测试 handler 不启动服务)
    test(web_root, [blocked]) :-
        % 需要 mock request，暂时 blocked
        true.

    %% c9: 数独求解
    test(sudoku_solve) :-
        Puzzle = [[5,3, _, _,7, _, _, _, _]
                 ,[6, _, _,1,9,5, _, _, _]
                 ,[_,9,8, _, _, _, _,6, _]
                 ,[8, _, _, _,6, _, _, _,3]
                 ,[4, _, _,8, _,3, _, _,1]
                 ,[7, _, _, _,2, _, _, _,6]
                 ,[_,6, _, _, _, _,2,8, _]
                 ,[_, _, _,4,1,9, _, _,5]
                 ,[_, _, _, _,8, _, _,7,9]],
        sudoku(Puzzle),
        Puzzle = [[5,3,4,6,7,8,9,1,2],
                  [6,7,2,1,9,5,3,4,8],
                  [1,9,8,3,4,2,5,6,7],
                  [8,5,9,7,6,1,4,2,3],
                  [4,2,6,8,5,3,7,9,1],
                  [7,1,3,9,2,4,8,5,6],
                  [9,6,1,5,3,7,2,8,4],
                  [2,8,7,4,1,9,6,3,5],
                  [3,4,5,2,8,6,1,7,9]].

    %% c5: Meta-interpreter
    :- use_module(meta_interpreter).

test(prove_true) :-
        prove(true).
    test(prove_atom, [setup(clear_m), cleanup(clear_m)]) :-
        assertz(m(foo)),
        prove(m(foo)).

:- end_tests(hello_pl_suite).

:- run_tests.
```

## 测试结果格式

plunit 默认输出简洁，加 `verbose` 可看详情：

```prolog
?- run_tests.
% PL-Unit: hello_pl_suite ......... done
% 9 tests passed
```

失败时：

```prolog
?- run_tests.
% PL-Unit: hello_pl_suite . f ....
% ERROR: .../test: test fib_base: assertion failed
% 8 tests passed, 1 failed
```

## 总结

- `:- begin_tests(Name)` / `:- end_tests(Name)` 定义测试集
- `test(Name)` 或 `test(Name, [Options])` 写测试用例
- Options: `fail`, `nondet`, `forall`, `setup`, `cleanup`, `blocked`, `time`
- `:- run_tests.` 自动执行
- 结合 CI 实现自动化测试
