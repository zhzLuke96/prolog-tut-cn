# 元编译器（Meta-interpreter）

> 一句话概括：用 Prolog 写一个解释 Prolog 的解释器——理解求值本质的最佳练习。

> 前置：[DCG](c4.md) | 难度：★★★ | 后续：[优化](c6.md), [lisprolog](../codeseen/cs2.md)

为什么要学这个？写元编译器不是为了在生产中替换 swipl，而是为了让你真正理解 Prolog 执行模型：深度优先怎么走、回溯怎么切、为什么要写纯谓词。把引擎内部翻出来，你就不会再被 "Prolog 怎么知道下一步做什么" 搞蒙了。

给前端开发者的类比：Babel plugin 遍历 AST 节点，你的 visitor 函数控制每个节点的解析路径。元编译器就是 Prolog 自己的 "Babel plugin API"。

## 为什么要学元编译器

用 Haskell 写编译器的人会告诉你：编译器是"把一种语言翻译成另一种语言的程序"。
用 Prolog 写元编译器的人会告诉你：元编译器是"用 Prolog 解释 Prolog 的程序"。

前端的读者可以把它类比成 Babel plugin——遍历 AST 节点，做转换，再输出。
后端的读者可以把它理解成"写一个自定义 DSL 的解释器"。

掌握 vanilla MI 后，你就能理解 Prolog 引擎背后的执行模型：谓词调用、回溯、cut 的底层机制，不再是黑魔法。

> 性能提醒：vanilla MI 在生产中通常有 10x-100x 性能开销，不适合作为运行时框架。

## 前置知识：clause/2

`clause/2` 是 Prolog 元编程的核心工具，也是 Vanilla Meta-interpreter 的基石——MI 每一层递归都靠它查找子句。

### 签名

```prolog
clause(+Head, -Body)
```

+Head 是已实例化的谓词（模式指示符），-Body 返回该子句的目标体。

### 事实与规则

- **事实**：没有子句体，`Body` 绑定为 `true`。

```prolog
?- clause(male(tom), Body).
Body = true.
```

- **规则**：子句体是逗号连接的目标列表，`Body` 返回联合体。

```prolog
?- clause(father(X, Y), Body).
X = _,
Y = _,
Body = (male(X), parent(X, Y)).
```

`listing/1` 内部就是用 `clause/2` 实现的——它遍历谓词的所有子句，逐个输出。用我们的例子：

```prolog
?- listing(father/2).
:- father(A, B) :-
    male(A),
    parent(A, B).
```

### 用途

- 元编程（生成、变换代码）
- 调试与追踪
- 自省动态谓词

更重要的是，**Vanilla Meta-interpreter 的核心循环就是 `clause/2`**：`prove(H) :- clause(H, Body), prove(Body).` 没有 `clause/2`，元编译器就无法获取程序本身的子句。

## 什么是元编译器

元编译器（Meta-interpreter）是用 Prolog 写的 Prolog 解释器。听起来套娃，但它的意义在于：

- 揭示 Prolog 执行模型（深度优先、从左到右、回溯）
- 允许定制执行策略（添加调试、计数、解释功能）
- 是实现规则引擎、专家系统的基础

经典参考：Sterling & Shapiro 《The Art of Prolog》第三章。

## Vanilla Meta-interpreter

最简的元编译器叫"vanilla meta-interpreter"，核心就三个子句：

```prolog
%% prove(+Goal) — 证明目标成立
prove(true) :- !.
prove((A, B)) :- !, prove(A), prove(B).
prove(H) :- clause(H, Body), prove(Body).
```

这就是 Prolog 执行模型的本质：

1. `true` 总是成功
2. `(A, B)` 先证明 A，再证明 B（从左到右）
3. 其他目标通过 `clause/2` 查找匹配子句，递归证明 Body

`clause/2` 是 ISO 标准谓词，`clause(Head, Body)` 返回匹配 Head 的子句，Body 是子句体。

### 试用

假设有：

```prolog
male(tom).
male(jerry).
parent(tom, jerry).
father(X, Y) :- male(X), parent(X, Y).
```

查询：

```prolog
?- prove(father(tom, jerry)).
true .

?- prove(father(jerry, X)).
X = tom .
```

注意：vanilla meta-interpreter 不做 cut 处理。`!/0` 在 Body 里被当作普通目标传给 `clause/2`，但 SWI-Prolog 的 `clause/2` 不会展开 cut——所以需要特殊处理。

## 扩展 1：Clause 计数

统计每个子句被调用的次数。用 `rule_used/2` 动态谓词记录计数器，每次命中子句时递增。

```prolog
:- dynamic rule_used/2.

prove_count(true, 0) :- !.
prove_count((A, B), N) :- !,
    prove_count(A, N1),
    prove_count(B, N2),
    N is N1 + N2.
prove_count(H, N) :-
    clause(H, Body),
    ( rule_used(H, N0) -> retract(rule_used(H, N0)), N1 is N0 + 1
    ; N1 = 1
    ),
    assert(rule_used(H, N1)),
    prove_count(Body, N2),
    N is N2 + 1.
```

查询后检查 `rule_used/2`：

```prolog
?- prove_count(father(tom, jerry), N).
N = 3 .

?- rule_used(Clause, Count).
Clause = male(tom),    Count = 1 ;
Clause = parent(tom, jerry), Count = 1 ;
Clause = father(tom, jerry), Count = 1 .
```

## 扩展 2：Debug 打印

进入和退出目标时分别打印，`Depth` 控制缩进层级。进入打印目标名，返回打印 `[ok]`。

```prolog
prove_debug(true, _) :- !.
prove_debug((A, B), Depth) :- !,
    prove_debug(A, Depth),
    prove_debug(B, Depth).
prove_debug(H, Depth) :-
    tab(Depth * 2),
    format("~w~n", [H]),
    clause(H, Body),
    D1 is Depth + 1,
    prove_debug(Body, D1),
    tab(Depth * 2),
    format("~w  [ok]~n", [H]).
```

输出：

```prolog
?- prove_debug(father(tom, jerry), 0).
father(tom, jerry)
  male(tom)          % Depth=1
  male(tom)  [ok]
  parent(tom, jerry)
  parent(tom, jerry) [ok]
father(tom, jerry) [ok]
true .
```

## 扩展 3：简单 Explainer

收集证明路径的结构化树。`prove_explain/3` 返回步骤列表，每步是 `step(Goal, SubSteps)` 节点。

```prolog
prove_explain(true, _, []) :- !.
prove_explain((A, B), Depth, Steps) :- !,
    prove_explain(A, Depth, Steps1),
    prove_explain(B, Depth, Steps2),
    append(Steps1, Steps2, Steps).
prove_explain(H, Depth, [step(H, SubSteps)]) :-
    clause(H, Body),
    D1 is Depth + 1,
    prove_explain(Body, D1, SubSteps).
```

有了步骤树，`explain/1` 将其渲染为缩进可读输出。

```prolog
explain(Goal) :-
    prove_explain(Goal, 0, Steps),
    print_steps(Steps, 0).

print_steps([], _).
print_steps([step(Goal, Subs)|Rest], Depth) :-
    tab(Depth * 2),
    format("~w  :-~n", [Goal]),
    D1 is Depth + 1,
    print_steps(Subs, D1),
    print_steps(Rest, Depth).
```

查询：

```prolog
?- explain(father(tom, jerry)).
father(tom, jerry)  :-
  male(tom)  :-
  parent(tom, jerry)  :-
true .
```

## 处理 Cut 和 Disjunction

真正完整的 meta-interpreter 还需要处理 `!/0` 和 `;/2`：

```prolog
prove(true) :- !.
prove(!) :- !.                 % TODO: 实际 cut 需要控制回溯
prove((A, B)) :- !, prove(A), prove(B).
prove((A; _)) :- !, prove(A).  % 只尝试第一个分支
prove((_; B)) :- !, prove(B).  % 回溯分支
prove(H) :- clause(H, Body), prove(Body).
```

完整的 cut 语义实现需要追踪 choice point，比较复杂，这里不展开。实际项目中用 `call/1` 替代即可。

## 总结

Meta-interpreter 是理解 Prolog 引擎的窗口。Vanilla 版本就三行，但揭示了：

- 子句查找（`clause/2`）是运行时核心
- 合取（`,`）对应顺序执行
- 回溯由 Prolog 引擎提供，meta-interpreter 只管递归

在此基础上添加计数、跟踪、解释功能，就是定制化推理引擎的雏形。
