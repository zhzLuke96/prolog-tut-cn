# Web Server

> 一句话概括：用 SWI-Prolog 内置 HTTP 库，几行代码启动一个 Web 服务器。

> 前置：c7 | 难度：★★ | 后续：—

## 背景

SWI-Prolog 自带全套 HTTP 库（`library(http/*)`），支持路由、JSON、Session、WebSocket。对内部工具或原型开发，比 Node.js/Flask 还省事——数据库、业务逻辑、HTTP 全在同一个语言里。

参考文件 `code/hello/websev.pl`：

```prolog
:- use_module(library(http/thread_httpd)).
:- use_module(library(http/http_dispatch)).
:- use_module(library(http/http_unix_daemon)).

:- http_handler(/, handle_request, []).

handle_request(_Request) :-
    format("Content-type: text/plain~n~n"),
    format("Hello!").
```

启动：

```prolog
?- http_server(handle_request, [port(8080)]).
% HTTP server started on port 8080
```

浏览器访问 `http://localhost:8080/` 输出 `Hello!`。

## http_handler 和 http_server

### http_server/2

启动 HTTP 服务：

```prolog
http_server(:Handle, +Options).
```

常用选项：

- `port(Port)` — 端口号，默认 4000
- `workers(N)` — 工作线程数
- `timeout(Secs)` — 连接超时

### http_handler/3

注册路由：

```prolog
http_handler(+Path, :Handle, +Options).
```

路由匹配：

```prolog
:- http_handler(/,          root_handler, []).
:- http_handler(/api/users, users_handler, []).
:- http_handler(/files/,    file_handler, [prefix]).
```

`[prefix]` 表示前缀匹配——`/files/foo.txt` 也路由到 `file_handler`。

### Request 对象

Handler 收到的 `Request` 是 Prolog 列表，包含：

```prolog
[ method(Method)
, path(Path)
, search_params(Params)
, ...
]
```

常用字段通过 `http_current_request/1` 和 `http_parameters/2` 提取：

```prolog
handle_request(Request) :-
    http_parameters(Request, [name(Name), age(Age, [number])]),
    format("Name=~w, Age=~d~n", [Name, Age]).
```

## 实战：JSON REST API

```prolog
:- use_module(library(http/thread_httpd)).
:- use_module(library(http/http_dispatch)).
:- use_module(library(http/http_json)).
:- use_module(library(http/json)).

:- http_handler(/,          root,         []).
:- http_handler(/api/echo,  echo_handler, []).
:- http_handler(/api/data,  data_handler, []).

server :-
    http_server(http_dispatch, [port(8080)]).

%% 根路由：HTML 欢迎页
root(_Request) :-
    reply_html_page(title("Prolog Web"),
        p("Prolog Web Server Running!")).

%% POST /api/echo — 接收 JSON，原样返回
echo_handler(Request) :-
    http_read_json_dict(Request, Dict),
    Reply = _{ status: ok, echo: Dict.get(data) },
    reply_json_dict(Reply).

%% GET /api/data — 返回结构化 JSON
data_handler(_Request) :-
    Data = _{ items: [_{id: 1, name: "foo"}, _{id: 2, name: "bar"}]
            , total: 2
            , timestamp: 1_700_000_000
            },
    reply_json_dict(Data, [status(200)]).

:- initialization(server).
```

保存为 `webserver.pl`，运行：

```bash
$ swipl -q webserver.pl
```

测试：

```bash
$ curl http://localhost:8080/
$ curl http://localhost:8080/api/data
$ curl -X POST -H "Content-Type: application/json" \
       -d '{"data":"hello"}' http://localhost:8080/api/echo
```

## 文件服务

推荐方式——直接用 `http_reply_from_dir/2` 一行搞定：

```prolog
:- use_module(library(http/http_server_files)).

:- http_handler('/static/', http_reply_from_dir('static'), [prefix]).
```

如果需要更精确的控制，也可以手动用 `http_attachment/3` 解析请求：

```prolog
:- use_module(library(http/http_dispatch)).
:- use_module(library(http/http_files)).

:- http_handler(/static/, serve_files, [prefix]).

serve_files(Request) :-
    http_attachment(Request, _File, [ unsafe(true) ]),
    http_reply_file('static/', [unsafe(true)], Request).
```

## 部署选项

### 后台运行

```prolog
:- use_module(library(http/http_unix_daemon)).

%% 启动脚本直接 daemon 化：
:- initialization(http_daemon).
```

`http_daemon` 自动 fork 到后台、写 PID 文件、处理日志。

### 嵌入到现有 Web 框架

SWI-Prolog 也能生成 CGI/FastCGI，但日常不推荐。

### HTTPS

SWI-Prolog HTTP 库支持 SSL：

```prolog
:- use_module(library(http/http_ssl_plugin)).

http_server(http_dispatch,
    [ port(443)
    , ssl(true)
    , cert_file('cert.pem')
    , key_file('key.pem')
    ]).
```

## 生产环境注意事项

- `worker/1` 设置合理线程数（默认 4）
- 用 `library(http/http_log)` 记录访问日志
- 用 `library(http/http_error)` 自定义错误页面
- Session 管理用 `library(http/http_session)`（默认关闭）

```prolog
:- use_module(library(http/http_session)).

handler(Request) :-
    http_session_id(SessionId),
    http_parameters(Request, [action(Action)]),
    ...
```

## 总结

SWI-Prolog 的 HTTP 库成熟且完整：

- `http_server/2` 启动服务，`http_handler/3` 注册路由
- `reply_json_dict/1` 方便返回 JSON
- `http_read_json_dict/2` 解析请求 JSON
- 静态文件、SSL、Session、daemon 模式全内置
- 适合内部 API、原型、小工具——一个 `.pl` 文件搞定全套

## Middleware 模式

> 进阶内容，可跳过

Middleware 是一种函数组合模式：每个 middleware 接收一个 handler，返回包装后的 handler。

### logger middleware

记录每个请求的 method 和 path：

```prolog
middleware_logger(Handler) :-
    Handler.

%% 实际 logger 通常在 handler 执行前后做额外操作
%% 这里是一个简化 wrapper
%% 完整实现需要 thread_httpd 下 hook 请求生命周期
```

### auth middleware

简单 token 检查——从 Authorization header 提取 bearer token：

```prolog
:- use_module(library(http/http_header)).

middleware_auth(Handler) :-
    Handler.

%% 实际检查逻辑需要在 handler 内部调用
%% 例如：
%% auth_wrapper(Request, Handler) :-
%%     (  http_header(Authorization, Bearer Token, Request)
%%     -> Handler(Request)
%%     ;  reply_json_dict(_{error: unauthorized}, [status(401)])
%%     ).
```

### rate_limit middleware

请求计数——使用 `nb_setval`/2 做简单计数器：

```prolog
middleware_rate_limit(Handler) :-
    Handler.

%% 示例计数逻辑：
%% rate_limit_check :-
%%     (  nb_current(request_count, Count)
%%     -> NewCount is Count + 1
%%     ;  NewCount = 1
%%     ),
%%     nb_setval(request_count, NewCount),
%%     (  NewCount > 100
%%     -> reply_json_dict(_{error: rate_limit_exceeded}, [status(429)])
%%     ;  true
%%     ).
```

### 组合 middleware

Middleware 通过嵌套组合：

```prolog
%% 实现 middleware 组合
compose_middleware([], Handler) :- Handler.
compose_middleware([MW|Rest], Handler) :-
    compose_middleware(Rest, Inner),
    call(MW, Inner).

%% 使用
:- http_handler(/, compose_middleware([middleware_logger, middleware_auth, middleware_rate_limit], handle_request), []).
```

### 局限

Prolog 没有函数式语言的 middleware 链天然表达力，上述实现是概念演示。生产环境建议使用 `library(http/http_hook)` 或 thread-local 变量实现横切关注点。

## 数据库操作（SQLite）

> 进阶内容，可跳过

SWI-Prolog 通过 `library(sqlite)` 提供 SQLite 绑定。

### 准备

```prolog
:- use_module(library(sqlite)).
```

### 连接与建表

```prolog
setup_db :-
    sqlite_open('todos.db', DB, []),
    sqlite_exec(DB, 'CREATE TABLE IF NOT EXISTS todos (
        id    INTEGER PRIMARY KEY AUTOINCREMENT,
        task  TEXT NOT NULL,
        done  INTEGER DEFAULT 0
    )'),
    sqlite_close(DB).
```

### CRUD 操作

```prolog
%% 创建 TODO
add_todo(DB, Task) :-
    sqlite_exec(DB, 'INSERT INTO todos (task) VALUES (?)', Task).

%% 查询所有 TODO
list_todos(DB, Todos) :-
    findall(todo{id:Id, task:Task, done:Done},
            (  sqlite_query(DB, 'SELECT id, task, done FROM todos', row(Id, Task, Done))
            ), Todos).

%% 标记完成
toggle_todo(DB, Id) :-
    sqlite_exec(DB, 'UPDATE todos SET done = CASE WHEN done THEN 0 ELSE 1 END WHERE id = ?', Id).

%% 删除 TODO
delete_todo(DB, Id) :-
    sqlite_exec(DB, 'DELETE FROM todos WHERE id = ?', Id).
```

### 集成到 Web 服务器

```prolog
:- use_module(library(http/thread_httpd)).
:- use_module(library(http/http_dispatch)).
:- use_module(library(http/http_json)).
:- use_module(library(sqlite)).
:- use_module(library(http/http_header)).

:- sqlite_open('todos.db', DB, []).

:- http_handler(/api/todos, todos_handler, []).

todos_handler(Request) :-
    http_parameters(Request, [method(M)]),
    (  M = get
    -> list_todos(DB, Todos),
       reply_json_dict(_{todos: Todos})
    ;  http_read_json_dict(Request, Body),
       add_todo(DB, Body.task),
       reply_json_dict(_{status: ok})
    ).
```

### 注意

- `sqlite_exec/3` 参数化查询用 `?` 占位，自动防 SQL 注入
- `sqlite_query/3` 返回 `row(...)` 结构；使用 `findall/3` 转换成 dict 方便 JSON 序列化
- 生产环境要管理连接池或每次请求独立连接

## 生产考量

> 进阶内容，可跳过

### CORS 设置

跨域请求需要手动添加 CORS header：

```prolog
:- use_module(library(http/http_header)).

cors_handler(Handler) :-
    Handler.

%% 在 handler 入口调用：
%% cors_set_headers :-
%%     http_set_response_header(access_control_allow_origin, '*'),
%%     http_set_response_header(access_control_allow_methods, 'GET, POST, PUT, DELETE, OPTIONS'),
%%     http_set_response_header(access_control_allow_headers, 'Content-Type, Authorization').
```

### Session 管理

使用 `library(http/session_manager)`：

```prolog
:- use_module(library(http/session_manager)).

%% 配置 session
:- initialization(http_session_manager(expire(3600))).

%% 在 handler 中使用
session_handler(Request) :-
    http_open_session(Request),
    http_session_data(user(User)),
    (  var(User)
    -> http_session_assert(user(anonymous))
    ;  true
    ).
```

### Request size limit

`http_server/2` 支持 `limit/1` 选项：

```prolog
http_server(http_dispatch,
    [ port(8080)
    , limit(10_000_000)  %% 限制请求体 10MB
    ]).
```

### 并发安全

- SWI-Prolog HTTP 服务器基于多线程（`workers(N)`）
- `nb_setval`/2 非线程安全；线程间共享数据用 `thread_local` + `with_mutex/2`
- `library(sqlite)` 实例默认线程安全，但多线程写需 `sqlite_exec` 同步

```prolog
:- thread_local request_counter/1.

increment_counter :-
    with_mutex(counter,
        (  request_counter(C),
           retractall(request_counter(_)),
           NC is C + 1,
           asserta(request_counter(NC))
        ;  asserta(request_counter(1))
        )).
```

### 部署建议

- 用 `swipl --daemon` 或 `http_daemon` 后台运行
- 反向代理到 nginx——nginx 处理 TLS、静态文件、负载均衡
- 使用 `library(http/http_log)` 记录访问日志
- 监控用 `library(http/http_monitor)` 或外部工具

```nginx
# nginx 配置片段
upstream prolog_backend {
    server 127.0.0.1:8080;
}
server {
    listen 80;
    location / {
        proxy_pass http://prolog_backend;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```
